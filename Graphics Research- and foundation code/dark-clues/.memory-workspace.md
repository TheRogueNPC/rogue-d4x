## Architecture Invariants

### Class Resolution
Haxe classes on window as flattened underscore names:
- `com.watabou.sevendrl.SevenDRL` ‚Üí `window.com_watabou_sevendrl_SevenDRL`
- Only available AFTER `lime.embed()` runs (synchronous, within the same call)

### No $extend, No $hxClasses
Both are IIFE-scoped inside SevenDRL.js. Custom inheritance uses `Object.create()`.

### No Random.Int
Use inline PRNG: `seed = seed * 48271.0 % 2147483647 | 0`
Or `ArrayExtender.random()`, `ArrayExtender.pick()`

### Hook Chain
Last registered hook wraps previous ones. `before` runs outside-in, `after` inside-out.
`this.__suppressOrig = true` stops chain.

### Grid Topology
`(h+1)√ó(w+1)` nodes at intersections, `h√ów` cells between them.
`grid.cell(i, j)` takes (row, col). Level Creator uses `[col, row]` (j, i).

## Level Creator (level-creator/index.html)

### Features
- Grid painting, room creation/deletion, hero/boss markers
- Character loadout (name, HP/stock bonus, theme) included in export
- **Encounters**: per-room enemy spawns loaded from `assets/enemies/manifest.json`
- Export: version, grid, rooms (cells, connections, doors, spawn), character
- Import: full restore from export JSON

### Enemy Integration
- Enemies defined in `assets/enemies/<name>.json` with `{ name, hp, damage, state, glyph }`

- Room's `spawn[]` array included in export/import
- Enemy data fetched asynchronously via XHR; cached in `ENEMY_CACHE`

## Rogue API Key Points

### Plan.create Override (unconditional)
`Plan.create` is overridden at rogue-api.js init to check `_customPlanData` on EVERY call. This means:
- Custom plan injection works even if `autoLoad()` hasn't resolved
- `_applyCustomPlan()` is one-shot (clears data after use)
- Hero repositioning + boss spawn on GameScene.reset after-hook

### Encounter System
- Hooks `hero.roomChanged` signal via GameScene.reset before-hook
- Definition-based (name, min/max level, room size, priority, once)
- Generic `onRoomEnter(fn)` handlers run before encounter checks
- `spawnInRoom()` helper picks unoccupied cells

### Cleanup
`Rogue.cleanup(preserveStock)` removes all mod DOM overlays by ID list:
`dc-pause-overlay`, `dc-difficulty-overlay`, `dc-intro-message`, `dc-aim`, `dc-spawn-indicator`, `dc-dev-toolbar`, `dc-char-creator-overlay`, `dc-chess-aim`, `dc-chess-indicator`, `dc-view-overlay`, `dev-console`, `dc-inspector`

Called automatically on `GameScene.reset` before-hook.
