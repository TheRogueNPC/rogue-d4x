# Scripting Reference — Automation Tools & Systems

Three new systems enable rapid, consistent feature development for true roguelike
complexity.

## System 1: Feature Generator Script

**Location:** `Dev-Tools/scripts/generate-feature.js`

**Purpose:** Scaffold working boilerplate for any feature type in ~30 seconds.

### Usage

```bash
node Dev-Tools/scripts/generate-feature.js <type> <args...>
```

### Supported Types

| Type | Arguments | Output |
|---|---|---|
| `mob` | `<name> <hp> <color> <info>` | `<name>-mob.js` |
| `ability` | `<name> <keyCode> <cost> <description>` | `<name>-ability.js` |
| `encounter` | `<name> <minLvl> <maxLvl> <roomSize> <desc>` | `<name>-encounter.js` |
| `pickup` | `<name> <type> <effect>` | `<name>-pickup.js` |

### Examples

```bash
# Goblin enemy: 2 HP, magenta, "A small goblin"
node Dev-Tools/scripts/generate-feature.js mob "Goblin" 2 0xFF00FF "A small goblin"

# IceBlast ability: key 'I' (73), cost 2 stock
node Dev-Tools/scripts/generate-feature.js ability "IceBlast" 73 2 "Freeze enemies"

# Spider nest encounter: levels 1-5, large rooms
node Dev-Tools/scripts/generate-feature.js encounter "spider_nest" 1 5 "large" "Arachnid lair"

# Health potion pickup: healing type, +2 HP effect
node Dev-Tools/scripts/generate-feature.js pickup "HealthPotion" "healing" "+2 HP"
```

### Workflow After Generation

1. **Customize:** Open the generated `.js` file, tweak values at `SWAP` points
2. **Register:** Uncomment in `Dev-Tools/mods/manifest.js`
3. **Reload:** `Ctrl+Shift+R` in browser
4. **Test:** Type `DBG.mobs()` (or relevant DBG command) to verify

### Generated File Structure

All generated files follow the same template pattern:

```js
// ═══════════════════════════════════════════════════════════════════════════════
// MOD: name.js — One-line description
// Load after: rogue-api.js
//
// This is a TEMPLATE — uncomment in manifest.js to activate.
// Generated by: node generate-feature.js <type> <args>
// ═══════════════════════════════════════════════════════════════════════════════

(function() {
    'use strict';

    // ╔══ SWAP POINT: Your customization here ────────────────────────

    Rogue.Mob.define({ ... });  // (or appropriate API call)

    if (window.devLog) window.devLog('[mod:name] loaded — ...');
})();
```

Every `SWAP POINT` comment marks a location you should customize.

## System 2: Progression System

**Location:** `Dev-Tools/mods/progression-system.js`

**Purpose:** Automatic difficulty scaling, ensuring challenge stays meaningful
across all levels.

### How It Works

- **Mob HP/Damage scale per level:** 1x at level 1 → 2x at level 10
- **Difficulty presets:** Easy (+2 HP), Normal (baseline), Hard (-1 HP), Nightmare (-2 HP)
- **Ability stock regeneration:** +0.5 stock/turn, but only after a 3-turn delay
- **Fully configurable** via `window.PROGRESSION` global

### Configuration

All settings live in `PROGRESSION` object at the top of the mod:

```js
var PROGRESSION = {
    mobHPScale: [0, 1, 1.1, 1.2, ..., 2.0],        // per level
    mobDamageScale: [0, 1, 1.1, 1.15, ..., 1.9],   // per level
    difficulty: {
        easy: { hpBonus: 2, stockBonus: 2 },
        normal: { hpBonus: 0, stockBonus: 0 },
        hard: { hpBonus: -1, stockBonus: -1 },
        nightmare: { hpBonus: -2, stockBonus: -2 }
    },
    stockRegenPerTurn: 0.5,
    stockRegenDelay: 3
};
```

### Tweaking in Console

```js
// View all settings
window.PROGRESSION

// Change mob HP scaling at level 5
window.PROGRESSION.mobHPScale[5] = 1.4

// Slower ability regeneration
window.PROGRESSION.stockRegenPerTurn = 0.3

// Add a custom difficulty
window.PROGRESSION.difficulty.custom = { hpBonus: 1, stockBonus: 1 }

// Query a value
window.getProgression(5, 'mobHPScale')  // returns 1.3 (level 5 mob HP multiplier)
```

### What Gets Scaled

- **Mob HP:** Automatically applied in Mob constructor
- **Hero bonuses:** Applied when difficulty is selected (before level starts)
- **Stock regen:** Applied each hero turn (automatic, no setup needed)
- **Mob damage:** Not automatically scaled (see below for custom hooks)

### Custom Damage Scaling (if needed)

Mob damage is typically 1-2; the game balances through HP more than damage.
If you want damage scaling too:

```js
ModLoader.hook('com.watabou.sevendrl.characters.StateChasing', 'engage', function() {
    var dmgScale = window.getProgression(window.game.level, 'mobDamageScale');
    var baseDamage = this.mob.damage || 1;
    var scaledDamage = Math.ceil(baseDamage * dmgScale);
    arguments[0] = scaledDamage;  // override damage
}, null);
```

## System 3: Content Pack System

**Location:** `Dev-Tools/mods/content-pack-system.js`

**Purpose:** Organize features into toggleable groups; allow players to
enable/disable packs; enable mods to check if their pack is active.

### Defining a Pack

In `content-pack-system.js`, add to `window.__contentPacks`:

```js
window.__contentPacks.bestiary = {
    enabled: true,  // true = ship enabled by default
    name: 'Bestiary Expansion',
    description: 'Wraiths, Goblins, and other creatures',
    features: ['wraith_mob', 'goblin_mob', 'zombie_mob']  // feature IDs
};
```

### Registering Features

Each feature needs a registration in `window.__contentFeatures`:

```js
window.__contentFeatures.wraith_mob = ['wraith-mob.js', 'wraith-encounter.js'];
window.__contentFeatures.goblin_mob = ['goblin-mob.js'];
```

Or, in the mod itself:

```js
// At the top of Dev-Tools/mods/wraith-mob.js:
if (!window.isFeatureEnabled('wraith_mob')) {
    if (window.devLog) window.devLog('[wraith-mob] disabled by content pack');
    return;  // don't register
}

// Register the feature:
Rogue.Mob.define({ name: 'Wraith', ... });
window.registerFeature('wraith_mob', ['wraith-mob.js']);
```

### Console Commands

```js
// List all packs
window.listContentPacks()

// Enable a pack
window.enablePack('bestiary')

// Disable a pack
window.disablePack('bestiary')

// Check if a feature is enabled
window.isFeatureEnabled('wraith_mob')

// Check if a pack is enabled
window.isPackEnabled('bestiary')

// Register a feature
window.registerFeature('my_feature', ['my-mod.js'])
```

### Use Case: Expanding Content

Ship the game with Core pack enabled (Wanderers, Hunters, basic stock system).
Add optional packs:

```
Core          ✓ (always on)
├─ Bestiary   ○ (enable: 10+ additional mobs)
├─ Arcana     ○ (enable: 8+ new abilities)
├─ Treasure   ○ (enable: 5+ new pickups)
└─ Challenges ○ (enable: bonus difficulty modes)
```

Players can enable as many or as few packs as they want.

## Integration Example: Add a Complete Feature Set

### Step 1: Generate and customize

```bash
# Create a Wraith mob
node Dev-Tools/scripts/generate-feature.js mob "Wraith" 3 0x8888FF "A spectral entity"

# Edit Dev-Tools/mods/wraith-mob.js
# - Change visual to 'W'
# - Customize damage to 2
# - Set HP 3 (already there)

# Create an encounter for them
node Dev-Tools/scripts/generate-feature.js encounter "wraith_lair" 3 8 "large" "Haunted necropolis"

# Edit Dev-Tools/mods/wraith-lair-encounter.js
# - Import Wraith class
# - Spawn 2-3 per encounter
```

### Step 2: Register in content pack

In `Dev-Tools/mods/content-pack-system.js`:

```js
window.__contentPacks.bestiary = {
    enabled: true,
    name: 'Bestiary Expansion',
    features: ['wraith_mob', 'wraith_encounter']  // ← add these
};

// At top of each mod, add:
if (!window.isFeatureEnabled('wraith_mob')) {
    if (window.devLog) window.devLog('[wraith-mob] disabled');
    return;
}
```

### Step 3: Enable in manifest

```js
// Uncomment both:
"/Dev-Tools/mods/wraith-mob.js",
"/Dev-Tools/mods/wraith-lair-encounter.js",
```

### Step 4: Test

```
Reload → DBG.mobs() → Wraith listed
Play → Enter a level 3-8 large room
→ Should encounter 2-3 Wraiths
```

### Step 5: Tune difficulty (if needed)

```js
// In console:
window.PROGRESSION.mobHPScale[5] = 1.4  // Wraiths get harder at level 5
```

## Workflow: 10 Mobs in 2 Hours

1. **Generate all 10:** ~5 min (30 sec × 10)
2. **Customize glyphs/colors:** ~20 min (2 min × 10)
3. **Create encounters for groups:** ~30 min (5 mobs per encounter)
4. **Register in pack system:** ~10 min
5. **Test & balance:** ~55 min (5-6 min × 10)

**Total:** ~2 hours for a complete "Bestiary Expansion" pack.
